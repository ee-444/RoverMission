Rotate in place to any given heading.

Rotate to any heading from 0-360. Will not move forward - just rotate. If the rover is in a infinate moving loop this will stop it once the heading is found.

[in]angle- heading you wish to face. Input is in degrees * 10 so 10 degrees is 100 and 300 degrees is 3000. this gives the resulution of 0.1 degrees

create a plan of attack

Create a distance to plaque and angle to plaque array for use in the map seek and destroy method. Based on constants such as ROOM_DIFFERENCE_AMOUNT, ROOM_CONSECUTIVE_CNT and MAX_PLAQUE_CNT find what is thought to be the left most edge of the plaque. REturns the distance in one array and angle from the last plaque in the other. elements are 0 if nothing is found

Parameters:
[in]map1- A map with data from the first scan (with or without plaques)
[in]map2- A map with data from the second scan (with or without plaques)
[out]heading_map- An array with the distance from home to each found plaque
[out]angle_map- An array with angles from home (and then the last plaque) to the next plaqe

Once at a plaque - find out how to get to the next.

Based on the distance and angle arrays - from any plaque the next one can be found. The next plaque will be the next plaque to the right. As did Zoolander - we can only turn right. This methods result make one assumption - that you are facing the plaque head on.

Parameters:
[in]dist- The array containing the distances to each plaque
[in]angle- The array containing the angles from one plaque to the next
[in]plaque_num- what plaque is next. Must be from 2 -> n.
[out]plaque_dist- the distance to the next plaque
[out]plaque_angle- If facing the plaque headon - this is the anle to the next plaque

uint16_t Mission::irDistance ( uint8_t  pin)

read the distance from a truncated and memory friendly FLASH table

Differences from the std 1024 distance array are that: 1. only values that are within an adjusted range are stored - logic returns otherwise 2. value is stored as 8 bits and multiplied by 2 when found. This uses 1/4 the FLASH as before

Parameters:
[in]pin- the analog pin that the sensor is connected to. - Makes a call to readADC
